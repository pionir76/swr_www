SmartRoute 는 임베디드 통신변환 디바이스이며, 다수의 범용 디바이스의 데이터를 통신으로 읽어와 내부 레지스터 테이블에 기록하고 이를 사용자 웹화면으로 실시간 보여주거나, Modbus TCP 프로토콜로 외부에 제공한다.

본 디바이스는 NXP 사의 I.MX6ULL CPU를 사용하며 개발 언어는 Qt 를 사용한다.

1. 범용 장비로부터 Modbus RTU/TCP 프로토콜을 통해 데이터를 읽어 들인다. 본 디바이스는 하나의 RS485 포트와 2개의 Ethernet switching hub 포트를 가지고 있다.

2. 본 디바이스는 다수의 장비에 RS485 / Ethernet 포트를 통해 한번에 연결할 수 있다.

3. 본 디바이스는 내부 웹서버(lighttpd)를 통해 정적 HTML/CSS/JS 파일을 제공하며, 웹 UI에서 발생하는 API 요청은 Qt App 내부의 QHttpServer가 처리한다. 장비 설정, 레지스터 조회, 실시간 데이터 등 모든 동적 데이터 처리는 Qt App이 SQLite DB를 직접 조회/갱신하여 응답한다.

4. 내부 웹서버를 통해 범용 장비를 등록할 수 있다. 예를 들어 RS485 통신을 할 것인지? Ethernet을 통해 연결할 것인지? 국번은 몇 번인지? IP 주소는 어떻게 되는지? 장비명은 무엇인지? 읽어 들이는 주기는 얼마인지? 타임아웃은 얼마인지 등등

5. 내부 웹서버를 통해 데이터 통신을 할 범용 장치의 레지스터들을 등록할 수 있다.

6. 수집된 다수의 범용 장치의 레지스터 값들은 내부 레지스터 테이블에 실시간 저장되고 갱신된다.

7. 사용자는 내부 웹서버에 접속해서 현재 읽어 들이고 있는 값들을 실시간 모니터링 할 수 있다.

8. 내부 레지스터 테이블에서 사용자는 항목을 선택해서 별도 트렌드 데이터로 기록 보관 할 수 있다. (레코딩 기능)

9. 사용자는 내부 웹서버에 접속해서 기록된 트렌드 데이터 항목들을 리스트로 확인할 수 있으며 트렌드 그래프로 열어 볼 수 있다.

10. 내부에 별도의 Modbus TCP 서버를 구축하여 사용자가 Modbus TCP 프로토콜로 원하는 내부 레지스터의 값을 읽어 갈 수 있도록 한다.

11. 사용자는 웹서버로 접속해서 자신만의 화면을 꾸밀 수 있다. 내부 레지스터 테이블에 있는 데이터를 웹페이지 컨트롤에 매핑하여 자신만의 화면으로 편집 가능하다. (미니 HMI 기능 제공)
