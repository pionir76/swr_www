# SmartRoute Web UI 레이아웃 구성 계획

## 1. 전체 UI 방향

SmartRoute 웹 화면은 단순한 장비 목록 화면이 아니라, **임베디드 통신 게이트웨이의 운영/설정/모니터링 콘솔**로 구성한다.

SmartRoute는 RS485 및 Ethernet으로 여러 장비의 데이터를 수집하고, 내부 RegisterTable에 실시간 저장한 뒤 웹 화면이나 Modbus TCP 서버로 제공하는 장치이다. 따라서 웹 UI는 **현장 엔지니어가 빠르게 상태를 보고, 장비를 등록하고, 통신 문제를 진단할 수 있는 관리형 대시보드**로 설계한다.

주요 목적은 다음과 같다.

```text
1. 현재 통신 상태를 한눈에 확인
2. 장비 등록/수정/삭제
3. 장비별 레지스터 등록/수정/삭제
4. 실시간 수집값 모니터링
5. 시스템 설정 및 사용자 관리
```

---

## 2. 기본 화면 구조

권장 레이아웃은 **좌측 사이드바 + 상단 상태바 + 본문 카드형 대시보드**이다.

```text
┌─────────────────────────────────────────────────────────────┐
│ Top Bar: SmartRoute / 현재 시간 / 로그인 사용자 / 상태요약  │
├───────────────┬─────────────────────────────────────────────┤
│ Sidebar       │ Main Content                                │
│               │                                             │
│ Dashboard     │ 페이지별 콘텐츠 영역                         │
│ Devices       │                                             │
│ Registers     │                                             │
│ Realtime      │                                             │
│ Trends        │                                             │
│ Modbus Server │                                             │
│ Settings      │                                             │
│ Users         │                                             │
│ Logs          │                                             │
└───────────────┴─────────────────────────────────────────────┘
```

i.MX6ULL 기반 임베디드 장비이므로 과한 애니메이션이나 무거운 프레임워크 느낌보다는, **가볍고 빠른 산업용 관리 화면** 느낌이 적합하다.

## 3. 주요 메뉴 구성

### 3.1 대시보드

첫 화면은 전체 상태 요약 중심으로 구성한다.

#### 상단 요약 카드

```text
[전체 장비 수] [정상 통신] [통신 오류] [활성 알람] [폴링 상태]
```

#### 본문 구성

```text
- 장비 통신 상태 요약
- RS485 Serial Worker 상태
- TCP Worker 상태
- 최근 오류 로그
- 실시간 주요 레지스터 Top 10
```

대시보드는 DB 자체보다 **DeviceList + RegisterTable + PollingManager 상태**를 보여주는 화면으로 잡는다.

---

### 3.2 장비 관리 화면

장비 관리 화면은 1차 개발 핵심 UI이다.

#### 레이아웃

```text
┌──────────────────────────────────────────────┐
│ 장비 관리                                     │
│ [장비 추가] [선택 삭제] [새로고침]             │
├──────────────────────────────────────────────┤
│ 검색: [장비명 / 코드 / IP / Slave ID]          │
├──────────────────────────────────────────────┤
│ 장비 목록 테이블                              │
│ 코드 | 장비명 | 연결 | 프로토콜 | IP | 국번 | 상태 │
├──────────────────────────────────────────────┤
│ 우측 또는 팝업: 장비 등록/수정 폼              │
└──────────────────────────────────────────────┘
```

#### 장비 등록/수정 폼 필드

```text
장비 코드        deviceCode     예: DEV-CHILLER-01
장비명           name           예: 냉동기 1호기
표시명           displayName    예: 냉동기 #1
연결 방식        serial / tcp
프로토콜         modbus_rtu / modbus_tcp / modbus_ascii / pclink
IP 주소          TCP 장비일 때만 활성
TCP 포트         기본 502
Slave ID         Modbus 국번
Timeout          ms
Polling Interval ms
Retry Count
Byte Order       big / little
```

#### 연결 방식별 폼 동작

```text
serial 선택 시:
- IP 주소 비활성
- TCP 포트 비활성
- Slave ID 활성

tcp 선택 시:
- IP 주소 활성
- TCP 포트 활성
- Slave ID 활성
```

---

### 3.3 레지스터 관리 화면

장비를 선택한 뒤 해당 장비의 레지스터 목록을 관리하는 화면이다.

#### 레이아웃

```text
┌──────────────────────────────────────────────┐
│ 레지스터 관리                                 │
├──────────────────────────────────────────────┤
│ 장비 선택: [냉동기 1호기 ▼]                    │
├──────────────────────────────────────────────┤
│ [레지스터 추가] [CSV 가져오기] [선택 삭제]      │
├──────────────────────────────────────────────┤
│ 태그명 | 표시명 | 타입 | 주소 | 길이 | 단위 | 스케일 | R/W │
└──────────────────────────────────────────────┘
```

#### 레지스터 추가/수정 폼

```text
태그명             tagName
표시명             displayName
레지스터 타입       coil / discrete_input / holding_register / input_register
주소               address
길이               length
통합 레지스터 ID    unifiedRegisterId
단위               unit
스케일             scale
최소값             minValue
최대값             maxValue
Byte Order         default / big / little
읽기 전용           readOnly
```

#### 레지스터 타입 도움말

```text
coil              → bit read/write
discrete_input    → bit read only
holding_register  → word read/write
input_register    → word read only
```

---

### 3.4 실시간 모니터링 화면

실시간 값은 `RegisterTable`을 조회하는 화면이다.

#### 레이아웃

```text
┌──────────────────────────────────────────────┐
│ 실시간 모니터링                               │
│ 자동 갱신: [ON] 주기 [1초 ▼]                  │
├──────────────────────────────────────────────┤
│ 필터: 장비 [전체 ▼] 상태 [전체 ▼] 검색 [     ] │
├──────────────────────────────────────────────┤
│ 장비명 | 태그명 | 표시명 | 현재값 | 단위 | 상태 | 갱신시각 │
└──────────────────────────────────────────────┘
```

#### 상태 표현

```text
정상        녹색
통신 오류    빨강
값 범위 초과 노랑
미갱신       회색
쓰기 대기    파랑
```

#### 값 표시 방식

```text
rawValue: 450
scaledValue: 45.0
unit: °C
```

기본 화면에서는 `scaledValue + unit`를 보여주고, 상세 펼침에서 raw 값을 보여주는 구성이 좋다.

---

### 3.5 폴링 제어 화면

폴링 제어는 독립 화면으로 두거나 대시보드 상단 카드에 포함할 수 있다.

#### 구성

```text
현재 폴링 상태: 실행 중 / 정지
Serial Worker: 실행 중 / 오류 / 장비 수
TCP Worker: 실행 중 / 오류 / 장비 수
[폴링 시작] [폴링 정지] [장비 설정 재로드]
```

RS485는 단일 버스 특성상 `SerialWorker`가 순차 폴링하고, TCP 장비는 장비별 독립 워커로 병렬 폴링하는 구조이므로, UI에서도 Serial/TCP 상태를 분리해서 보여준다.

---

### 3.6 시스템 설정 화면

시스템 설정은 3개 탭으로 나눈다.

```text
[네트워크] [RS485] [시스템]
```

#### 네트워크 탭

```text
eth0 / eth1
사용 여부
역할: field / service
모드: static / dhcp
IP Address
Netmask
Gateway
[적용] [취소]
```

#### RS485 탭

```text
Device: /dev/ttymxc1
Baudrate
Data bits
Parity
Stop bits
[적용]
```

#### 시스템 탭

```text
Hostname
NTP 사용 여부
시간 정보
로그 레벨
앱 재시작
공장 초기화
DB 백업/복원
```

네트워크 설정, 재시작, 초기화는 위험한 기능이므로 관리자 권한에서만 보이게 한다.

---

### 3.7 사용자 관리 화면

사용자 모델은 ID, 사용자명, 비밀번호 해시·솔트, 권한을 포함한다.

#### 레이아웃

```text
사용자 목록
- ID
- 사용자명
- 표시명
- 권한
- 활성 여부
- 마지막 로그인
- 관리 버튼
```

#### 권한

```text
일반사용자
매니저
관리자
```

#### 권한별 UI 노출 계획

```text
일반사용자:
- 대시보드
- 실시간 모니터링
- 트렌드 조회

매니저:
- 장비/레지스터 설정 가능
- 폴링 제어 가능

관리자:
- 사용자 관리
- 시스템 설정
- DB 초기화
- 재시작
```

---

### 3.8 로그 화면

개발 및 현장 진단에 필요한 화면이다.

#### 구성

```text
로그 필터: INFO / WARN / ERROR
검색어
최근 500줄 보기
[새로고침] [다운로드]
```

#### 표시 항목

```text
시간 | 레벨 | 메시지
```

통신 실패, DB 오류, API 오류, 폴링 상태 변화가 보이면 현장에서 문제 파악이 쉬워진다.

## 4. 2차 기능 화면 계획

2차 기능은 별도 메뉴로 두되, 1차 UI에서 메뉴만 비활성 상태로 미리 배치해도 좋다.

### 4.1 트렌드 화면

SmartRoute는 선택한 내부 레지스터 값을 별도 트렌드 데이터로 기록하고, 사용자가 리스트와 그래프로 볼 수 있도록 계획한다.

#### 레이아웃

```text
트렌드 항목 목록
- 기록명
- 장비명
- 레지스터명
- 저장 주기
- 보존 기간
- 기록 상태

그래프 영역
- 기간 선택
- 확대/축소
- CSV 다운로드
```

---

### 4.2 미니 HMI 빌더

사용자는 내부 레지스터 테이블의 데이터를 웹 컨트롤에 매핑해 자신만의 화면을 구성할 수 있다.

#### 레이아웃

```text
좌측: 위젯 팔레트
- 숫자 표시
- 램프
- 게이지
- 버튼
- 텍스트
- 차트

중앙: 캔버스
우측: 속성 패널
- 매핑 레지스터
- 표시명
- 단위
- 색상 조건
- 최소/최대값
```

## 6. 디자인 톤 제안

SmartRoute는 산업용 통신 변환 장치이므로 디자인은 아래 방향이 적합하다.

```text
스타일: 산업용 대시보드 / 네트워크 장비 관리 콘솔
배경: 밝은 회색 또는 다크 모드 선택 가능
포인트 컬러: 파랑 또는 청록
상태 컬러:
- 정상: 녹색
- 경고: 노랑
- 오류: 빨강
- 미연결: 회색
- 동작 중: 파랑
```

카드와 테이블 위주의 UI가 적합하다.

```text
대시보드: 카드 + 상태 배지
장비 관리: 테이블 + 우측 편집 패널
레지스터 관리: 테이블 + 모달 편집
실시간 모니터링: 고밀도 테이블
설정 화면: 탭 + 폼
```

## 7. 화면 흐름

사용자 관점의 기본 흐름은 다음과 같다.

```text
로그인
  ↓
대시보드에서 전체 상태 확인
  ↓
장비 관리에서 장비 등록
  ↓
레지스터 관리에서 읽을 항목 등록
  ↓
폴링 시작
  ↓
실시간 모니터링에서 값 확인
  ↓
필요 시 트렌드 기록 또는 HMI 화면 구성
```

이 흐름이 1차 목표인 “핵심 수집 기능 + 기본 웹 관리 UI 완성”과 잘 맞는다.

## 9. 결론

SmartRoute 웹 UI는 단순 설정 화면보다 **장비 통신 상태를 실시간으로 보여주는 산업용 관리 콘솔**로 설계하는 것이 좋다.

1차 시안의 핵심은 다음 4개이다.

```text
1. 대시보드
2. 장비 관리
3. 레지스터 관리
4. 실시간 모니터링
```

여기에 시스템 설정, 사용자 관리, 로그 화면을 붙이면 현장 테스트와 제품화에 필요한 기본 UI가 갖춰진다.
