# SmartRoute 사용자 관리 정책 가이드

## 1. 목적

이 문서는 SmartRoute Industrial Gateway System의 웹 관리 화면에서 사용할 사용자 계정, 권한, 로그인 보안, 계정 상태 관리 정책을 정의한다.

SmartRoute는 설비 통신 게이트웨이이므로 일반 웹 서비스보다 **운영 안정성**, **설정 변경 통제**, **접근 권한 분리**, **감사 추적**을 중요하게 다룬다.

---

## 2. 사용자 계정 관리 기본 원칙

사용자 계정은 시스템에 접속하여 장비 상태 조회, 설정 변경, 로그 확인, 유지보수 작업을 수행하기 위한 인증 단위이다.

기본 원칙은 다음과 같다.

- 모든 사용자는 고유한 사용자 ID를 가진다.
- 사용자 ID는 중복될 수 없다.
- 사용자 ID는 생성 후 변경하지 않는 것을 원칙으로 한다.
- 비밀번호는 관리자도 직접 확인할 수 없고, 변경 또는 초기화만 가능해야 한다.
- 사용하지 않는 계정은 삭제보다 비활성화 처리를 우선한다.
- 시스템 설정, 사용자 관리, 펌웨어 업데이트 등 위험 작업은 관리자 권한으로 제한한다.

---

## 3. 사용자 권한 등급

SmartRoute의 사용자 권한은 3단계로 구분한다.

| 권한 | 표시명 | 설명 |
|---|---|---|
| `ADMIN` | 관리자 | 시스템 전체 관리 권한 |
| `MANAGER` | 매니저 | 장비 및 운영 설정 관리 권한 |
| `USER` | 일반사용자 | 조회 중심의 제한 권한 |

---

## 4. 권한별 기능 접근 범위

### 4.1 일반사용자 (`USER`)

일반사용자는 시스템 상태를 조회하는 계정이다.

허용 기능:

- 대시보드 조회
- 실시간 모니터링 조회
- 통합 레지스터 값 조회
- 트렌드 조회
- 일부 로그 조회

제한 기능:

- 장비 추가/수정/삭제
- 레지스터 설정 변경
- 폴링 설정 변경
- 시스템 설정 변경
- 사용자 관리
- 파라메터 백업/복원
- 펌웨어 업데이트
- 시스템 재시작

---

### 4.2 매니저 (`MANAGER`)

매니저는 설비 운영자가 사용하는 계정이다.

허용 기능:

- 대시보드 조회
- 실시간 모니터링 조회
- 장비 관리
- 레지스터 관리
- 폴링 설정 관리
- HMI 설정 관리
- 트렌드 조회
- 로그 조회
- 일부 시스템 운영 설정 변경

제한 기능:

- 사용자 계정 관리
- 로그인 보안 정책 변경
- 펌웨어 업데이트
- 공장 초기화
- 시스템 중요 설정 변경
- 관리자 계정 수정

---

### 4.3 관리자 (`ADMIN`)

관리자는 시스템 전체를 관리하는 계정이다.

허용 기능:

- 전체 메뉴 접근
- 사용자 계정 생성/수정/비활성화
- 권한 변경
- 비밀번호 초기화 또는 변경
- 로그인 보안 정책 변경
- 네트워크/시스템 설정 변경
- 파라메터 백업/복원
- 펌웨어 업데이트
- 시스템 재시작
- 로그 및 감사 이력 확인

관리자 권한은 최소 인원에게만 부여하는 것을 원칙으로 한다.

---

## 5. 계정 상태 정책

사용자 계정 상태는 다음 3가지로 관리한다.

| 상태 | 표시 | 의미 |
|---|---|---|
| `ACTIVE` | 활성 | 정상 로그인 가능 |
| `DISABLED` | 비활성 | 관리자가 계정을 사용 중지한 상태 |
| `LOCKED` | 잠김 | 보안 이벤트로 인해 임시 로그인 차단된 상태 |

---

## 6. ACTIVE 상태

`ACTIVE`는 정상 사용 가능한 계정이다.

조건:

- 계정이 활성화되어 있음
- 로그인 실패 횟수가 허용 범위 이내임
- 계정 잠금 시간이 만료되었거나 잠금 상태가 아님
- 관리자가 비활성화하지 않음

---

## 7. DISABLED 상태

`DISABLED`는 관리자가 계정을 사용하지 못하도록 비활성화한 상태이다.

사용 사례:

- 퇴사자 계정
- 임시 작업자 계정 종료
- 장기간 사용하지 않는 계정
- 보안상 사용 중지가 필요한 계정

정책:

- `DISABLED` 상태의 계정은 로그인할 수 없다.
- 로그인 실패 횟수와 관계없이 접근이 차단된다.
- 관리자가 다시 `ACTIVE`로 변경해야 사용할 수 있다.
- 계정 삭제보다 비활성화를 우선한다.

---

## 8. LOCKED 상태

`LOCKED`는 계정이 삭제되거나 비활성화된 것이 아니라, 보안상 로그인만 차단된 상태이다.

대표적인 발생 조건:

- 비밀번호 연속 실패 횟수 초과
- 관리자가 수동으로 계정을 잠금
- 의심스러운 로그인 시도 감지
- 짧은 시간 내 반복 로그인 실패

기본 정책:

| 항목 | 기본값 |
|---|---|
| 로그인 실패 허용 횟수 | 5회 |
| 잠금 해제 방식 | 관리자 수동 해제만 가능 |

동작 예시:

1. 사용자가 비밀번호를 잘못 입력한다.
2. `failed_login_count`가 1씩 증가한다.
3. 실패 횟수가 5회 이상이면 계정 상태를 `LOCKED`로 변경한다.
4. `LOCKED` 상태는 자동으로 해제되지 않는다.
5. 관리자가 상태를 `ACTIVE`로 변경해야만 로그인이 가능하다.

---

## 9. DISABLED와 LOCKED의 차이

| 구분 | DISABLED | LOCKED |
|---|---|---|
| 원인 | 관리자 조치 | 보안 이벤트 또는 관리자 조치 |
| 성격 | 장기 사용 중지 | 로그인 차단 |
| 자동 해제 | 없음 | 없음 |
| 관리자 해제 | 가능 | 가능 |
| 대표 사례 | 퇴사자, 임시 계정 종료 | 비밀번호 5회 실패 |

정리하면 다음과 같다.

- `DISABLED`: 관리자가 의도적으로 꺼둔 계정
- `LOCKED`: 보안 문제로 인해 임시로 잠긴 계정

---

## 10. 로그인 보안 정책

로그인 보안 설정 화면에서는 다음 항목을 관리한다.

| 항목 | 설명 | 권장 기본값 |
|---|---|---|
| 로그인 실패 허용 횟수 | 계정 잠금 전 허용되는 실패 횟수 | 5회 |
| 세션 타임아웃 | 미사용 상태에서 자동 로그아웃까지의 시간 | 30분 |
| 비밀번호 최소 길이 | 비밀번호 최소 문자 수 | 8자 |
| 자동 로그아웃 | 세션 타임아웃 사용 여부 | 사용 |

> 참고: 계정 잠금 해제는 자동 해제 없이 관리자 수동 해제만 지원하므로 잠금 시간 항목은 존재하지 않는다.

---

## 11. 비밀번호 정책

비밀번호는 사용자가 직접 입력하지만, 시스템은 원문을 저장하지 않아야 한다.

정책:

- 비밀번호 원문 저장 금지
- 해시 값만 저장
- 관리자도 기존 비밀번호 확인 불가
- 관리자 기능은 “비밀번호 확인”이 아니라 “비밀번호 변경” 또는 “초기화”로 제공
- 최소 길이 기본값은 8자로 설정
- 가능하면 영문/숫자/특수문자 조합 권장

운영 초기에는 복잡한 비밀번호 정책보다 다음 정도를 우선 적용한다.

- 최소 8자 이상
- 사용자 ID와 동일한 비밀번호 금지
- 빈 비밀번호 금지
- 관리자 계정 기본 비밀번호 변경 유도

---

## 12. 세션 정책

세션은 로그인 후 사용자의 인증 상태를 유지하는 단위이다.

권장 정책:

- 기본 세션 타임아웃: 30분
- 사용자가 일정 시간 조작하지 않으면 자동 로그아웃
- 로그아웃 시 세션 무효화
- 관리자 권한 기능 실행 전 세션 유효성 재확인
- 가능하면 브라우저 종료 시 세션 만료

산업용 현장에서는 모니터링 화면을 장시간 켜두는 경우가 많으므로, 향후에는 조회 전용 화면과 설정 화면의 세션 정책을 분리할 수 있다.

예:

- 모니터링 화면: 긴 세션 허용
- 설정 변경 화면: 짧은 세션 또는 재인증 요구

## 13. 사용자 관리 화면 구성

사용자 설정 화면에는 다음 항목을 포함한다.

| 컬럼 | 설명 |

| 권한 | 관리자, 매니저, 일반사용자 |

### 13.4 선택 사용자 정보 패널

포함 항목:

- 설명

- 저장 버튼
- 비활성화 버튼

### 13.5 로그인 보안 설정 패널

포함 항목:

- 로그인 실패 허용 횟수
- 계정 잠금 시간
- 세션 타임아웃
- 비밀번호 최소 길이
- 자동 로그아웃 사용 여부

### 13.6 최근 로그인 이력

포함 항목:

- 시간
- 사용자 ID
- 결과
- IP 주소

## 15. DB 필드 권장안

사용자 테이블에 필요한 기본 필드는 다음과 같다.

```sql
CREATE TABLE users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    username TEXT NOT NULL UNIQUE,
    display_name TEXT NOT NULL,
    password_hash TEXT NOT NULL,
    role TEXT NOT NULL,
    status TEXT NOT NULL,
    description TEXT,
    failed_login_count INTEGER DEFAULT 0,
    locked_until TEXT,
    last_login_at TEXT,
    last_login_ip TEXT,
    created_at TEXT NOT NULL,
    updated_at TEXT NOT NULL
);
```

권장 role 값:

```text
ADMIN
MANAGER
USER
```

권장 status 값:

```text
ACTIVE
DISABLED
LOCKED
```

로그인 이력 테이블 예시:

```sql
CREATE TABLE login_history (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    username TEXT NOT NULL,
    result TEXT NOT NULL,
    ip_address TEXT,
    message TEXT,
    created_at TEXT NOT NULL
);
```

## 16. 로그인 처리 흐름

### 16.1 로그인 성공

```text
1. 사용자 ID 조회
2. 계정 상태 확인
3. 비밀번호 검증
4. failed_login_count 초기화
5. locked_until 초기화
6. last_login_at 저장
7. last_login_ip 저장
8. 로그인 성공 로그 기록
9. 세션 생성
```

### 16.2 로그인 실패

```text
1. 사용자 ID 조회
2. 계정 존재 여부 확인
3. 비밀번호 실패 시 failed_login_count 증가
4. 실패 횟수가 허용 횟수 이상이면 LOCKED 처리
5. locked_until 저장
6. 로그인 실패 로그 기록
```

### 16.3 잠긴 계정 로그인 시도

```text
1. status가 LOCKED인지 확인
2. LOCKED 상태이면 로그인 차단 (자동 해제 없음)
3. 관리자가 ACTIVE로 변경해야만 로그인 가능
```

## 17. 관리자 기능 정책

관리자가 수행할 수 있는 사용자 관리 기능은 다음과 같다.

- 사용자 추가
- 사용자 정보 수정
- 사용자 권한 변경
- 사용자 상태 변경
- 비밀번호 변경 또는 초기화
- 잠금 해제
- 계정 비활성화
- 로그인 보안 정책 변경

주의 사항:

- 자기 자신의 관리자 권한을 실수로 제거하지 못하도록 제한하는 것이 좋다.
- 마지막 남은 관리자 계정은 비활성화할 수 없도록 해야 한다.
- 관리자 계정 삭제는 제공하지 않는 것이 안전하다.
- 기본 관리자 계정의 초기 비밀번호는 반드시 변경하도록 유도한다.

## 20. 기본 정책 요약

SmartRoute 사용자 관리의 기본 정책은 다음과 같다.

```text
권한은 ADMIN / MANAGER / USER 3단계로 구분한다.
계정 상태는 ACTIVE / DISABLED / LOCKED 3단계로 관리한다.
비밀번호 5회 연속 실패 시 계정을 LOCKED 처리한다.
LOCKED 상태는 자동으로 해제되지 않으며, 관리자가 수동으로 ACTIVE로 변경해야 한다.
DISABLED 상태는 관리자가 다시 활성화하기 전까지 로그인할 수 없다.
관리자 계정은 최소화하고, 마지막 관리자 계정은 비활성화할 수 없도록 한다.
비밀번호 원문은 저장하지 않고 해시 값만 저장한다.
사용자 관리 및 로그인 관련 이벤트는 감사 로그로 남긴다.
```
